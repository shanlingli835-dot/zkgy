# Pattern catalog

Patterns describe how multiple Implemented components form a repeatable user flow or site
structure. They are not substitutes for page content or permission to invent new components.

## Current status

No pattern is `Implemented` yet. Do not treat the names below as generation instructions.

| Pattern            | Status  | Intended scope                                                 |
| ------------------ | ------- | -------------------------------------------------------------- |
| Page shell         | Planned | Header, main landmark, footer, skip link, responsive container |
| Primary navigation | Planned | Desktop navigation, dropdowns, mobile menu, active state       |
| Marketing CTA      | Planned | One primary action with optional secondary navigation          |
| Feedback           | Planned | Alert, validation, loading, blocking confirmation              |
| Forms              | Planned | Labels, inputs, errors, actions, success behavior              |
| Product detail     | Planned | Hero, capabilities, proof, related navigation, CTA             |

Create a pattern only after every component it uses is Implemented. Pattern documents must state
content order, primary action, responsive behavior, keyboard order, and public-copy boundaries.
