# ZKSC Website Rebuild — EXPERIENCE.md

## Foundation

本 UX 规范面向单站点响应式企业官网，覆盖桌面端、平板端和移动端。`DESIGN.md` 是视觉身份来源；本文件定义信息架构、页面模板、组件行为、状态、响应式和验收规则。

本次重构是源站 UI 重构，不是内容战略重写。所有页面范围、页面模块、文案含义、导航链接、CTA 路径、SEO 和素材迁移要求必须回查 PRD 与 `docs/source-site/page-inventory.md`。如果 mockup、视觉草稿或实现代码与本文件冲突，以本文件和 `DESIGN.md` 为准。

## Information Architecture

| Surface | Reached from | Purpose |
|---|---|---|
| 首页 | `/index.html` / logo / 导航首页 | 建立公司定位、核心安全能力、产品矩阵、解决方案、案例和 CTA |
| 产品详情页 | 首页产品入口 / 产品导航 | 展示单个产品定位、功能、优势、集成能力和咨询入口 |
| 解决方案页 | 首页方案入口 / 解决方案导航 | 展示行业场景、方案概述、方案组成、价值优势、业务场景和咨询入口 |
| 技术服务页 | 服务导航 | 展示渗透测试服务说明、服务优势、检测内容和联系入口 |
| 公司介绍页 | 关于导航 / 首页关于模块 | 展示公司介绍、研发实力、文化、荣誉、地区、FAQ 和联系入口 |
| 联系页及其他旧链接 | CTA / 导航 | 本次不重构，但链接必须保留并可达 |

主导航必须包含产品、解决方案、技术服务、公司介绍、联系我们等源站路径。移动端导航使用折叠菜单，不能丢失二级产品和解决方案入口。

## Voice and Tone

品牌 voice 在 `DESIGN.md`。页面微文案规则如下：

| Do | Don't |
|---|---|
| 使用“咨询方案”“免费试用”“联系我们”等源站 CTA 文案 | 为了统一按钮而改写业务含义 |
| 保留专业术语：GB 44495-2024、UNECE R155、SBOM、DevSecOps、TARA、LLM 等 | 把技术术语改成泛泛营销词 |
| 模块标题短而具体，正文解释能力和场景 | 添加“颠覆行业”“极致赋能”等空泛表达 |
| 异常/缺失状态直接说明原因和下一步 | 使用内部实现说明或开发备注作为外部文案 |

## Page Template Rules

### 首页模板

首页从上到下保持源站业务顺序：

1. 首屏：公司定位、低空与商业航天网络安全方案、主 CTA。
2. 合作生态/可信证明：客户、院校、行业 logo 或成果数据。
3. 核心安全能力：协议安全、智能安全、供应链安全、代码安全、应用安全、数据安全。
4. 核心产品介绍：六个产品入口保持可扫描。
5. 行业解决方案：网联汽车、特种装备入口和“了解更多”。
6. 关于公司：公司介绍、研发实力、发展信息。
7. 成功案例：保留源站案例模块。
8. 底部 CTA：免费试用/联系入口。

首页不做只看起来炫的首屏。首屏必须同时回答“公司是谁、解决什么安全问题、下一步去哪”。

### 产品详情页模板

六个产品页共享模板，但每页保留产品差异：

1. 产品首屏：产品名、定位描述、主 CTA。
2. 为什么需要/为什么选择：对应源站的场景或价值解释。
3. 产品功能：用能力块或 tab 展示，但移动端必须纵向展开。
4. 产品优势：用对比、指标、标准、检测覆盖或报告能力表达。
5. DevSecOps/集成能力：保留源站集成模块和相关素材。
6. 底部 CTA：免费试用或联系我们。

产品页禁止把不同产品改成同一套空泛卖点。`Wisdom`、`Swift AI`、`Swift Code`、`Wisdom Lens`、`Swift SCA`、`Swift Pentest` 的能力重点必须分别保留。

### 解决方案页模板

解决方案页必须让行业客户先看到场景，再看到工具链：

1. 方案首屏：行业名称、实验室/实验能力定位、咨询 CTA。
2. 方案概述：法规、标准、工具链和建设目标。
3. 方案组成：分组呈现工具、平台、实验区、数据管理等内容。
4. 价值优势：工具即合规、自动化、自主可控、低成本等。
5. 业务场景：tab 或纵向分组呈现具体检测/验证场景。
6. 底部 CTA：咨询方案。

网联汽车页必须保留 GB 44495-2024、UNECE R155、TARA、固件/协议/整车/云端验证。特种装备页必须保留脆弱性、渗透性、生存性、自主可控实验能力和整改闭环。

### 技术服务页模板

渗透测试服务页以“服务是什么、怎么做、为什么可信、怎么联系”为主线：

1. 服务首屏：渗透测试服务定位和联系 CTA。
2. 服务说明：面向整车及零部件的测试价值。
3. 服务优势：专业团队、全流程服务、高效全面。
4. 检测内容：保留源站检测模块。
5. 底部 CTA：联系我们。

### 公司介绍页模板

公司介绍页以可信度为主，不变成招聘页或品牌故事页：

1. 首屏：公司定位和联系 CTA。
2. 关于公司：保留源站公司介绍文本含义。
3. 研发实力/数据：专利、研发占比、团队信息等。
4. 企业文化：创新、融合、奋斗、幸福。
5. 荣誉资质：保留源站资质素材。
6. 公司分布地区：总部和城市信息。
7. FAQ：常见问题展开/收起。
8. 底部 CTA：联系我们/免费试用。

## Component Patterns

| Component | Use | Behavioral rules |
|---|---|---|
| Global Navigation | 全站 | 桌面端 hover/点击展开二级菜单；移动端折叠菜单覆盖全部导航项；当前页状态可见 |
| CTA Button | 首屏、模块尾、底部 CTA | 主 CTA 使用 `{components.button-primary}`；次 CTA 使用 `{components.button-secondary}`；目标链接保持源站路径 |
| Product Summary | 首页产品介绍 | 点击进入对应产品详情页；同一行/同一组内高度稳定，不因文案长短造成跳动 |
| Capability Block | 产品功能和核心能力 | 标题、说明、图标/截图三者至少保留标题和说明；不要仅显示图标 |
| Solution Tabs | 解决方案业务场景 | 桌面端可使用 tab；移动端改为纵向分组或 accordion；禁止 hover-only |
| Metric Block | 首页、服务、公司介绍 | 数字和说明文字绑定，滚动动画不得污染标题层级或可访问文本 |
| FAQ Item | 公司介绍页 | 点击整行展开；键盘可操作；展开状态可识别 |
| Footer Link Group | 页脚 | 保留备案、外链、站内链接；移动端按组折行，不隐藏 |

## State Patterns

| State | Surface | Treatment |
|---|---|---|
| Default load | 所有页面 | 首屏文字和主 CTA 先可见，图片渐进加载 |
| Navigation open | 桌面/移动 | 背景保持可读；焦点限制在菜单内；Esc 或再次点击关闭 |
| CTA hover/focus | 所有 CTA | 视觉反馈清楚；focus ring 不被移除 |
| Product tab active | 产品/方案模块 | active 状态使用 `{components.tab-active}`；inactive 仍可读 |
| Image missing | 图片/素材 | 保留布局空间，显示中性占位和 alt；不让模块坍塌 |
| Long title | 产品页/方案页 | 自动换行，不缩放到不可读，不遮挡 CTA |
| Mobile menu | 移动端 | 全部导航项可触达，二级项不依赖 hover |
| External link | 页脚/社区链接 | 保留目标，建议新窗口打开并有可识别标签 |

## Interaction Primitives

- Click/tap 是主要交互；hover 只能作为增强，不作为唯一入口。
- Tab、accordion、FAQ、导航菜单必须支持键盘访问。
- 动效只用于状态转换、滚动进入、hover 反馈，时长控制在 120-240ms。
- 禁止无限滚动、复杂 3D、全屏遮挡动画、纯装饰粒子效果。
- 页面内部锚点必须滚动到正确模块，且导航高度不会遮挡标题。
- 所有 CTA 点击必须可追溯到源站盘点中的链接。

## Accessibility Floor

- 文本和背景对比度达到 WCAG AA。
- 交互控件触控区域不小于 44px。
- 移动端正文最小 14px，主要正文建议 16px。
- 焦点顺序与视觉阅读顺序一致。
- 图片必须有 alt；纯装饰图 alt 为空但不能承载必要信息。
- Reduce Motion 开启时，滚动/数字/进入动效应关闭或降级。
- FAQ、tab、菜单必须有正确的展开状态和键盘操作状态。

## Responsive & Platform

| Breakpoint | Behavior |
|---|---|
| `>= 1200px` | 12 栅格；内容宽度 1200-1280px；产品/能力可 2-3 列，但避免卡片墙 |
| `1024-1199px` | 仍保持桌面导航；复杂图文模块降为 2 列 |
| `768-1023px` | 导航可折叠；模块以单列或 2 列为主；方案 tab 检查可触控 |
| `< 768px` | 单列；导航抽屉；按钮满宽或双按钮纵向；tab 改 accordion/分组 |

移动端不允许横向滚动。长中文标题必须换行，英文产品名不能撑破容器。

## Inspiration & Anti-patterns

- **可以借鉴**：高质量 B2B 安全/开发者工具网站的信息层级、产品架构图表达、产品截图嵌入方式、稳定导航和清楚 CTA。
- **不可照搬**：参考站的颜色、图标、精确构图、品牌图形。
- **拒绝**：PPT 式卡片网格、纯黑白交替区段、无意义大面积渐变、带文字的生成图片、和业务无关的抽象素材。

## Key Flows

### Flow 1 — 企业客户从首页进入方案咨询

1. 访客打开首页。
2. 首屏看到公司定位、低空与商业航天网络安全能力和主 CTA。
3. 继续浏览核心安全能力和行业解决方案。
4. 点击“网联汽车”或“特种装备”了解更多。
5. 在解决方案页查看方案概述、组成、价值优势和业务场景。
6. **Climax:** 点击“咨询方案”，进入 `contact.html`，路径与源站一致。

### Flow 2 — 技术负责人评估产品能力

1. 访客从导航进入产品菜单。
2. 选择 Swift SCA、Swift Code、Swift AI 等产品。
3. 产品页首屏说明产品定位和适用场景。
4. 向下查看产品功能、优势、集成能力。
5. **Climax:** 访客能区分该产品与其他产品的能力重点，并通过“免费试用/联系我们”进入咨询。

### Flow 3 — 访客确认公司可信度

1. 访客从导航进入公司介绍。
2. 查看公司介绍、研发数据、企业文化、荣誉资质和地区分布。
3. 打开 FAQ 了解解决方案、报价、试用等问题。
4. **Climax:** 访客通过 FAQ 或底部 CTA 找到联系路径。
